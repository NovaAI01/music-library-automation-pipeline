# Evidence Reliability Engine

Evidence Reliability Engine v1 is a reusable, read-only scoring layer for
metadata evidence. It helps downstream normalization, album cohesion, metadata
suggestions, and future learning distinguish canonical evidence from polluted
source text.

## Scoring

Each record receives:

- `reliability_score`: float from `0.0` to `1.0`
- `reliability_tier`: `high`, `medium`, or `low`
- `reliability_flags`: deterministic pattern flags
- `rationale`: short review snippets explaining the score

The engine starts from a neutral evidence score, subtracts for pollution, and
adds weight for corroboration. Scores are clipped to the 0.0-1.0 range.

## Evidence Weighting

Reliability increases when evidence has repeated support:

- repeated canonical agreement
- normalization knowledge support
- album cohesion support
- repeated folder consistency
- sequential track support
- prior approved review decisions
- low observed conflict rate

Reliability decreases when evidence shows conflict or source pollution:

- uploader or channel signatures
- label channel signatures
- `official audio`, `official video`, or visualizer wording
- remaster, deluxe, or anniversary edition noise
- embedded platform branding
- excessive separators
- all-caps casing anomalies
- mixed artist naming patterns
- repeated non-musical tokens
- suspicious bracket or suffix repetition
- artist mismatch against folder context
- autogenerated or placeholder source names

## Pollution Detection

The engine intentionally avoids giant blacklists. It uses compact, explainable
pattern families for uploader/channel artifacts, platform remnants, naming
noise, and canonical agreement. These flags are evidence for review and
weighting, not instructions to remove metadata.

## Downstream Usage

Album cohesion uses reliability to down-rank polluted album names and discount
low-reliability uploader artifacts where possible. Metadata suggestions include
reliability score, flags, and rationale snippets so reviewers can see whether a
suggestion came from stable evidence or noisy source text.

Generate reports with:

```bash
python -m app.main evidence-reliability --out reports
```

Outputs:

```text
reports/evidence_reliability/
  evidence_reliability_summary.json
  unreliable_evidence.csv
  reliable_patterns.csv
  reliability_groups.json
```

The review UI is available at `/review/reliability`.

## Safety Boundaries

The reliability engine does not:

- mutate media files
- write tags
- move, delete, or quarantine files
- auto-remove metadata
- call external APIs
- train or run ML models
- auto-approve future suggestions

It only reads existing reports and local review knowledge, then writes review
reports for human inspection and deterministic downstream weighting.
